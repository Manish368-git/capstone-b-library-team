## Activity 2 — Health Check Endpoint

A `/healthz` endpoint was implemented to verify that the backend service is operational.

Example response:

```
{
  "status": "ok",
  "version": "alpha",
  "time": "2026-03-12T12:03:14"
}
```

This endpoint allows automated tools and CI pipelines to confirm system health.

---

## Activity 3 — Logging

Application logging was enabled in the backend server.
The logs capture key request information including:

* HTTP request method
* endpoint path
* response status code

Example log entry:

```
GET /healthz 200
```

These logs help developers monitor system activity and debug issues during staging.

---

## Activity 4 — CI Smoke Test

A CI smoke test workflow was configured using GitHub Actions.
The workflow runs automatically on pull requests and verifies that the application starts successfully and responds correctly to the `/healthz` endpoint.

This prevents faulty code from being merged into the main branch.

---

## Activity 5 — Demo Preparation

A short demo script was prepared to present the staging environment and system health monitoring features.
The demonstration includes:

* showing the deployed staging URL
* accessing the `/healthz` endpoint
* displaying server logs
* explaining the CI validation process

The script is saved in:

```
docs/demo_script.md
```
